#  [](https://rds-storybook.azurewebsites.net/?path=/docs/components-button-overview--docs#button)Button
OverviewGuidelinesAPI
A **Button** is an essential interface element that enable users to perform an action or command by clicking or tapping on them. They can be used for tasks such as completing a process or submitting information.
**Buttons** are vital interface elements that allow users to perform actions or commands with a click or tap. They can be utilized for tasks such as completing a process or submitting information.
There are five options for button types:
  * **Primary Button** - Used for page primary action.
  * **Secondary Button** - Used alongside primary to reduce visual noise when there are many actions of equal importance on the page, or to support without overpowering for less critical tasks.
  * **Tertiary Button** - The Tertiary button, with its subtle and transparent background, is used for low-priority actions that require a softer approach. It is very important to only utilize the Tertiary button on solid background designs with our primary and secondary backgrounds. Avoid using this button when overlaying map data or imagery.
  * **Danger Button** - Used to signal actions that require caution or may have significant consequences, such as deletions or irreversible changes.
  * **Ghost Button** - Used for less prominent actions where visual clutter needs to be minimized. It can also be used inline because it differs from content in style and is recognizable as a button alongside content. Its hover state adds a nice level of interactivity and visual flair if used properly and sparingly.

##  [](https://rds-storybook.azurewebsites.net/?path=/docs/components-button-overview--docs#design-usage)Design Usage
**When to use:**
  * Triggering an action.
  * Enabling a "final" action.
  * Progressing or regressing a user through a step in a flow.
  * Submitting requested information.
  * Confirming the completion of a flow or cancelling out of it.
  * Using tag="a" and href when navigating to another page on the site.


**When not to use :**
  * Taking users to a different part within the same page, instead, use links.


##  [](https://rds-storybook.azurewebsites.net/?path=/docs/components-button-overview--docs#content)Content
**Do:**
  * Use sentence case, e.g. This is sentence case.
  * Omit punctuation, e.g. ! ? .
  * Use the button label to set expectations for what happens when a button is selected.
  * Begin the label text with an action verb followed by an object receiving the action.


**Don't:**
  * Use all caps.
  * Use long or redundant button text.
  * Use icons for decoration.
  * Add non-essential words.
  * Include the action verb when the button is part of navigation or group of buttons, and the verb is repeated.
