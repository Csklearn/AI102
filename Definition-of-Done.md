# Definition of Done / PR Requirements

- PR description has been updated based on the work done
- AC has been met
- Scope creep has been avoided 
  - PR comments that are out of scope need a new PBI created
- When developing, we are empowered to make code improvements we see fit
- Our PR checklist has been reviewed + met
- Appropriate documentation relating to the work has been updated
- Appropriate testing + accessibility has been met
- Dev team has signed off (through 2 approvers on PR)
- QA team has signed off 
- PR has been directed into the agreed release branch
